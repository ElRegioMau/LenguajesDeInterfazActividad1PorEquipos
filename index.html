Este es un ejemplo de problema

Objetivo

Desarrollar una aplicación web funcional utilizando control de versiones con GitHub. El proyecto se realizará en equipos de 4 estudiantes y cada integrante trabajará en una parte diferente del sistema utilizando ramas, commits y pull requests.

Requisitos del Proyecto

  • Sistema de Login
  • Uso de al menos una API externa (ejemplo: clima, IA, noticias, etc.)
  • Dashboard con información dinámica
  • Base de datos
  • Interfaz web funcional
  • Uso obligatorio de GitHub para manejo de versiones
  • Resolver conflictos

Ideas de Aplicación (los alumnos pueden elegir otra)

  • Dashboard del clima
  • App de noticias
  • App de seguimiento de hábitos
  • Administrador de tareas
  • Dashboard de criptomonedas
  • App de recomendaciones de películas

Arquitectura sugerida

Frontend

Backend

Base de datos

Flujo de Trabajo con Git

  1. Un estudiante crea el repositorio en GitHub.
  2. Agrega a sus compañeros como colaboradores.
  3. Cada estudiante clona el repositorio.
  4. Cada estudiante crea una rama para su trabajo.
  5. Se realizan commits frecuentes con mensajes claros.
  6. Se hacen Pull Requests para integrar cambios.
  7. El equipo revisa el código antes de fusionar cambios.

Asignación de Roles

Alumno 1: Backend y Login

  • Crear la estructura del servidor backend.
  • Implementar el sistema de login (usuario y contraseña).
  • Crear endpoint de autenticación.
  • Conectar el backend con la base de datos.
  • Crear rutas para obtener datos del usuario.

Alumno 2: Base de Datos

  • Diseñar el esquema de base de datos.
  • Crear tablas necesarias (usuarios, datos de la app, etc).
  • Implementar consultas básicas.
  • Integrar la base de datos con el backend.
  • Crear scripts para insertar datos de prueba.

Alumno 3: API Externa

  • Investigar y seleccionar una API pública.
  • Realizar peticiones a la API desde el backend o frontend.
  • Procesar los datos recibidos.
  • Mostrar la información en el dashboard.
  • Manejar posibles errores de conexión o datos.

Alumno 4: Frontend y Dashboard

  • Diseñar la interfaz web.
  • Crear página de login.
  • Crear dashboard principal.
  • Mostrar datos provenientes de la API.
  • Aplicar estilos CSS y mejorar la experiencia de usuario.

Buenas Prácticas de Git

  • Realizar commits frecuentes.
  • Usar mensajes claros en cada commit.
  • No trabajar directamente en la rama main.
  • Crear ramas para cada funcionalidad.
  • Resolver conflictos de manera colaborativa.

Ejemplos de mensajes de commit

git commit -m "Agrega sistema de login"
git commit -m "Conecta API de clima"
git commit -m "Crea diseño inicial del dashboard"
git commit -m "Agrega esquema de base de datos"
