# LenguajesDeInterfazActividad1PorEquipos

Proyecto Colaborativo Web — Manejo de Versiones con Git
Descripción

Este proyecto tiene como objetivo desarrollar una aplicación web colaborativa utilizando control de versiones con Git y trabajo en equipo mediante GitHub.

El proyecto será desarrollado por equipos de 4 estudiantes, donde cada integrante será responsable de una parte específica del sistema. El propósito principal es practicar un flujo de trabajo profesional de desarrollo de software utilizando repositorios, ramas, commits y pull requests.

Durante el desarrollo del proyecto los estudiantes aprenderán a:

Clonar repositorios
Crear y trabajar con ramas
Realizar commits con mensajes claros
Integrar cambios mediante Pull Requests
Resolver conflictos de versiones
Colaborar en el desarrollo de una aplicación real
Requisitos del Proyecto

La aplicación web debe incluir al menos los siguientes componentes:

Sistema de Login de usuarios
Consumo de una API externa (por ejemplo clima, IA, noticias, etc.)
Dashboard que muestre información dinámica
Conexión a una base de datos
Interfaz web funcional
Uso obligatorio de GitHub para control de versiones

Los equipos pueden elegir libremente el tipo de aplicación que desean desarrollar.

Ejemplos de aplicaciones posibles:

Dashboard del clima
Aplicación de noticias
Administrador de tareas
App de seguimiento de hábitos
Dashboard de criptomonedas
Recomendador de películas
Organización del Equipo

Cada equipo está formado por 4 integrantes, con las siguientes responsabilidades principales:

Alumno 1 – Backend y autenticación

Crear servidor backend
Implementar sistema de login
Crear endpoints para autenticación
Conectar backend con la base de datos

Alumno 2 – Base de datos

Diseñar esquema de base de datos
Crear tablas necesarias
Implementar consultas
Integrar base de datos con el backend

Alumno 3 – Integración de API

Investigar una API pública
Realizar peticiones a la API
Procesar los datos obtenidos
Integrar los datos en el sistema

Alumno 4 – Frontend y dashboard

Diseñar interfaz web
Crear página de login
Desarrollar dashboard
Mostrar datos dinámicos en pantalla
Flujo de Trabajo
Un integrante crea el repositorio.
Se agregan los demás integrantes como colaboradores.
Cada integrante clona el repositorio.
Cada integrante crea su propia rama para trabajar.
Se realizan commits frecuentes.
Se crean Pull Requests para integrar cambios.
El equipo revisa y aprueba los cambios antes de fusionarlos.
